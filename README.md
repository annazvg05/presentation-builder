# Генератор презентаций на основе ИИ

Создаёт брендированные PPTX-презентации из текстового ТЗ через Claude Code.

*(вставь скриншот готовой презентации)*

---

## Главное преимущество

Слайды собираются не из произвольно сгенерированных моделью кусков, а из шаблона, который воспроизводит структуру и оформление, принятые в презентациях Zerocoder. Например, кейс-слайд выглядит именно так, как он выглядит в реальных материалах компании — с теми же карточками этапов, цветами, иконками и расположением элементов.

Это не «презентация, сгенерированная нейросетью» — это презентация в уже узнаваемом фирменном стиле, просто собранная быстрее.

Дополнительное удобство именно такого подхода — в том, что всё хранится в одном месте:
- **Библиотека иконок пополняется за секунду** — достаточно добавить PNG в папку `Иконки/`, и она сразу доступна для любой следующей презентации
- **Фотографии — туда же**: добавил в `Фото/` и больше не думаешь об этом
- **Готовый файл появляется автоматически** в папке `Готовые презентации/` с названием из ТЗ — ничего не нужно переименовывать и скачивать вручную
- **Дизайн-система живёт в одном файле** (`CLAUDE.md`) — поменял правило один раз, и оно применяется ко всем новым слайдам

---

## Когда это нужно

Основной способ делать презентации — руками в PowerPoint или Google Slides, это остаётся. Генератор существует для случаев, когда дизайн-ресурса нет или сроки горят: нужно быстро собрать кейс-слайд, добавить биографию спикера или подготовить материалы к вебинару — тогда за пару минут можно получить готовый файл через генератор.

---

## Как это работает

Архитектура двухшаговая, чтобы результат был предсказуемым:

```
ТЗ.txt  →  Claude Code (скилл)  →  build.py  →  Готовые презентации/название.pptx
```

**Шаг 1 — Интерпретация ТЗ.** Текст задания передаётся в Claude Code через скилл `presentation-builder`. Модель определяет типы слайдов, извлекает заголовки, тексты, этапы, подбирает иконки по смыслу — и обновляет `build.py`. Модель ничего не придумывает сама: она только структурирует то, что явно написано в ТЗ.

**Шаг 2 — Сборка PPTX.** Дальше работает чистый Python: данные подставляются в заранее выверенные функции-шаблоны (`build_section_slide`, `build_keis_slide`, `build_prompts_slide` и т.д.). Никакой генерации вёрстки моделью — структура, цвета, отступы и типографика детерминированы и зафиксированы в дизайн-системе.

## Возможности

- Генерация полной презентации по тексту ТЗ одним запросом
- Поддержка разных типов слайдов из коробки:

| Тип слайда | Описание |
|---|---|
| **Секционный** | Один крупный текст на ярком градиенте — для разделов и заголовков |
| **Кейс с этапами** | Бейдж + заголовок + ситуация + три карточки с иконками |
| **Примеры промптов** | Три горизонтальные карточки с иконкой, заголовком и текстом |
| **Сравнение** | Две колонки: плюсы и минусы с иконками |
| **Биография спикера** | Фото + имя + роль + структурированные блоки с подзаголовками |
| **Галерея с фото** | Несколько фотографий в ряд + текстовый блок |

- Автоматический подбор иконки по смыслу контекста из папки `Иконки/`
- Фотографии всегда со скруглёнными углами и точным сохранением пропорций
- Текст разбивается на смысловые блоки с подзаголовками — без монотонных простыней
- Минимальные отступы между блоками соблюдаются автоматически

## Структура проекта

```
📁 Презентация/
├── build.py                    ← скрипт генерации, обновляется под каждое ТЗ
├── CLAUDE.md                   ← дизайн-система: цвета, шрифты, позиции, правила
├── logo.svg                    ← логотип (рендерится автоматически через Pillow)
├── ТЗ.txt                      ← текущее техзадание
│
├── 📁 Иконки/                  ← добавляй PNG сюда — сразу доступны для всех презентаций
├── 📁 Фото/                    ← фотографии спикеров, вузов и других объектов
├── 📁 Готовые презентации/     ← сюда автоматически сохраняется результат
│
└── 📁 .claude/skills/
    └── presentation-builder/
        └── skill.md            ← инструкция для Claude Code
```

## Запуск

Понадобится Python 3 и Claude Code CLI.

```bash
pip3 install python-pptx Pillow
```

Запуск после обновления `build.py`:

```bash
python3 build.py
```

Готовый файл появится в `Готовые презентации/` с названием из ТЗ.

---

## Как пользоваться

1. **Написать ТЗ** в файл `ТЗ.txt` — описать слайды в свободной форме. Если в тексте есть пометка `(фото)` — положить нужное изображение в папку `Фото/` заранее

2. **Открыть Claude Code** в папке проекта и вызвать скилл:
   ```
   /presentation-builder
   ```

3. **Получить файл** — Claude обновит `build.py` и запустит его. Презентация появится в `Готовые презентации/<название>.pptx`

4. **Добавить иконку** — просто положить PNG в `Иконки/`. При следующем запросе Claude подберёт её автоматически по смыслу

*(вставь скриншот процесса в Claude Code)*

## Google Slides (опционально)

При желании можно подключить Google Drive API — тогда после генерации Claude сразу выдаст ссылку на презентацию в Google Slides для онлайн-редактирования, без ручной загрузки файла.

## Итог

Генератор не заменяет дизайнера и не пытается «придумать» презентацию за вас. Он делает другое: берёт то, что вы написали в ТЗ, и собирает из этого аккуратный, визуально последовательный файл — быстро и без ручной работы в редакторе. Дизайн-система зафиксирована, иконки и фото подключаются в один клик, результат предсказуем.

Проект создан в рамках курса [Zerocoder](https://zerocoder.ru) с использованием Claude Code.
