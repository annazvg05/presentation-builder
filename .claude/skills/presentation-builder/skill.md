# Presentation Builder — zerocoder.ru

## Что делает этот скилл

Получает ТЗ от пользователя → обновляет `build.py` → запускает `python3 build.py` → готов файл в `готовые презентации/`.

Все правила дизайна описаны в `CLAUDE.md` (в папке проекта).  
Скрипт находится в `build.py` (в той же папке).

---

## Алгоритм при новом ТЗ

1. Прочитать `CLAUDE.md` (там полная дизайн-система и API функций)
2. Прочитать текущий `build.py` чтобы понять состояние
3. Установить `PRESENTATION_NAME` в build.py — имя из названия ТЗ без префикса "ТЗ. "  
   Пример: "ТЗ. Обратная связь" → `PRESENTATION_NAME = "Обратная связь"`
4. Определить типы слайдов по ТЗ:
   - Один крупный текст → `build_section_slide(prs, "Текст")`
   - Кейс с этапами → функция кейса из build.py
   - Примеры промптов → функция промптов из build.py
   - Сравнение двух вариантов → функция сравнения из build.py
   - Биография спикера → функция bio из build.py
   - Кастомный → писать новую функцию по правилам CLAUDE.md
5. Добавить/обновить функции в `build.py`
6. Обновить `main()` — слайды в правильном порядке
7. Запустить `python3 build.py` из папки проекта
8. Сообщить результат — файл в `готовые презентации/<название>.pptx`


## Быстрая справка по дизайн-системе

### Цвета фона
| Слайд | Градиент |
|---|---|
| Обычный | `#14133D → #2C2265 → #423388` (сверху вниз) |
| Секционный / заголовочный | `#5638B2 → #A493F9` (диагональ) |

### Ключевые размеры шрифта
| Элемент | Размер |
|---|---|
| Заголовок слайда (кейсы, кастомные) | 36pt Bold |
| Секционный заголовок | 54pt Bold |
| Ситуация (инлайн) | 20pt |
| Подзаголовки внутри блоков | 16pt Bold White |
| Текст карточек этапов | 16pt |
| Заголовок карточки промпта | 16pt Bold |
| Тело карточки промпта | 16pt |
| Текст в колонках сравнения | 16pt |
| Акцентный текст (роль, статус) | 17pt Bold — цвет ACCENT |
| Мелкий биографический текст | 14pt Secondary |
| Предупреждение | 12pt |

Правило: если текст не помещается — уменьшай шрифт на 2pt. Наложений быть не должно.

## Позиции на кейс-слайдах (дюймы)

| Элемент | y | h |
|---|---|---|
| Бейдж | 0.50" | 0.30" |
| Заголовок 36pt | 1.02" | 1.40" |
| Ситуация 20pt | 2.55" | **1.10"** (запас под 3+ строки) |
| Карточки этапов | **3.82"** | 1.68" |
| Предупреждение | 5.65" | 0.44" |

Ситуация может занимать 3+ строк — блок всегда делать h=1.10", карточки не раньше 3.82".

---

## Отступы — обязательные правила

- **Минимум 0.14"** (~10px) между любыми смысловыми блоками
- Подзаголовок + тело блока — единая группа, без зазора внутри
- Между группами — всегда отступ 0.14"
- Текстовый блок никогда не должен визуально касаться следующего элемента
- Если контент не помещается — уменьшай шрифт или объединяй пункты, но не сокращай отступы

## Работа с текстом — структура и типографика

Если в ТЗ есть перечисления или текст можно разбить по смыслу — **обязательно делать блоки**:
- Каждый смысловой раздел начинается с **подзаголовка** (16pt Bold White)
- Под ним — тело раздела (14–16pt Secondary)
- Между блоками — отступ 0.14"

Пример для биографии:
```
[Подзаголовок] "Обучаю сотрудников"   ← 16pt Bold White
[Тело]  "Газпром, РЖД, РосАтом..."    ← 14pt Secondary
[отступ 0.14"]
[Подзаголовок] "Образование"          ← 16pt Bold White
[Тело]  "Мехмат МГУ, MBA Kingston..." ← 14pt Secondary
```

## Фотографии

**Пропорции — строгое правило:** никогда не задавать w и h одновременно произвольно.  
Всегда вычислять один размер из реального aspect ratio изображения:

```python
from PIL import Image
img = Image.open(path)
w_px, h_px = img.size
# Фиксируем ширину, считаем высоту:
ph_w = int(3.60 * IN)
ph_h = int(ph_w * h_px / w_px)
# Или фиксируем высоту, считаем ширину:
ph_h = int(4.80 * IN)
ph_w = int(ph_h * w_px / h_px)
```

**Скругление обязательно** — использовать `_round_image(path)` перед `add_picture()`.  
Функция уже есть в `build.py`.

Если фото нужно центрировать по вертикали:
```python
ph_y = (SH - ph_h) // 2
```

## Выбор иконок

Библиотека иконок пополняется — всегда смотри актуальный список в `иконки/`.

Выбирай по смыслу контекста:
- **Результат, плюсы, достижения** → галочка
- **Минусы, риски, ошибки, отрицание** → крестик (всегда для "Минусы:")
- **Сбор данных, первый этап, действие** → молния
- **Анализ, инсайт, что-то крутое** → огонёк
- **Документы, отчёты, бизнес** → портфель
- **Команда, сотрудничество, партнёрство** → рукопожатие
- **Локация, города, регионы** → локатор (если есть)
- **Время, дедлайны** → часы (если есть)

Если подходящей иконки нет — используй ближайшую по смыслу из доступных.  
Обращай внимание на спецсимволы в именах файлов (например, неразрывный пробел `\xa0`).

## Технические ограничения

- Фон **только** через `_apply_gradient()` — `slide.background` не работает в Keynote/Google Slides
- Иконки в карточках — **left-aligned**, не по центру
- `add_chrome(slide)` — вызывать на каждом слайде (зелёная полоска + футер + логотип)
- Текстовые блоки не клипируются в PPTX — если текст больше блока, он вылезет визуально. Всегда давать высоту с запасом.
