# Presentation Builder — Полный гайд для Claude Code

## Что это за проект

Генератор брендированных PPTX-презентаций в стиле zerocoder.ru.  
Архитектура: `ТЗ.txt → Claude Code (этот скилл) → build.py → Готовые презентации/<название>.pptx`

Репозиторий: https://github.com/annazvg05/presentation-builder  
Рабочая папка: `/Users/anna/Desktop/Презентация/`

## 1. Создание / обновление презентации по ТЗ

### Алгоритм

1. Прочитать `CLAUDE.md` — там полная дизайн-система и сигнатуры всех функций
2. Прочитать `ТЗ.txt` — если не указан явно, спросить у пользователя
3. Прочитать текущий `build.py` — понять, что уже есть
4. Установить `PRESENTATION_NAME` в `build.py` (имя из ТЗ без префикса "ТЗ. ")  
   Пример: `"ТЗ. Обратная связь"` → `PRESENTATION_NAME = "Обратная связь"`
5. Определить типы слайдов и написать/обновить соответствующие функции
6. Обновить `main()` — слайды в правильном порядке
7. Запустить: `python3 build.py`
8. Сообщить результат — путь к файлу в `Готовые презентации/`

### Типы слайдов

| Функция | Когда использовать |
|---|---|
| `build_section_slide(prs, "Текст")` | Один крупный текст на ярком градиенте — разделы, заголовки |
| `build_keis_slide(...)` | Бейдж + заголовок + ситуация + три карточки с иконками |
| `build_prompts_slide(...)` | Три горизонтальные карточки с иконкой, заголовком и текстом |
| `build_comparison_slide(...)` | Две колонки: плюсы и минусы с иконками |
| Кастомный | Писать новую функцию по правилам из `CLAUDE.md` |

### Выбор иконок

Смотри актуальный список: `ls Иконки/`

| Смысл | Иконка |
|---|---|
| Результат, плюсы, достижения | `галочка.png` |
| Минусы, риски, ошибки | `крестик.png` |
| Сбор данных, первый этап, действие | `молния.png` |
| Анализ, инсайт | `огонек.png` |
| Документы, отчёты, бизнес | `портфель\xa0— средний размер.png` |
| Команда, сотрудничество | `рукопожатие.png` |

> Обращай внимание на неразрывный пробел `\xa0` в имени файла портфеля.

### Фотографии — правило пропорций

Никогда не задавать w и h одновременно произвольно. Всегда вычислять один из реального aspect ratio:

```python
from PIL import Image
img = Image.open(path)
w_px, h_px = img.size
ph_w = int(3.60 * IN)
ph_h = int(ph_w * h_px / w_px)   # считаем высоту из ширины
```

Скругление: всегда использовать `_round_image(path)` перед `add_picture()`.

## 3. Диагностика и исправление ошибок

### Типичные ошибки и решения

**FileNotFoundError на иконке или фото**  
→ Проверь точное имя файла: `ls Иконки/` или `ls Фото/`  
→ Помни про `\xa0` в имени портфеля

**Текст вылезает за пределы блока**  
→ Увеличь высоту текстового блока `h` или уменьши размер шрифта на 2pt  
→ В PPTX текст не клипируется — если блок мал, текст видно за его пределами

**Элементы перекрываются**  
→ Пересчитай `y`-координаты: минимальный зазор между блоками — 0.14"  
→ Используй позиции из CLAUDE.md как эталон

**Ошибка импорта `lxml` или `pptx`**  
→ `pip3 install python-pptx lxml Pillow`

**Фон не отображается в Keynote/Google Slides**  
→ Убедись, что фон задаётся через `_apply_gradient()`, а не через `slide.background`

### Как отлаживать

```bash
# Запустить с выводом ошибки
python3 build.py 2>&1

# Проверить синтаксис без запуска
python3 -m py_compile build.py && echo "Syntax OK"
```

## 4. Внесение изменений в дизайн-систему

Все правила дизайна живут в **`CLAUDE.md`** — это единственный источник истины.

Если нужно изменить цвет, шрифт, отступ или добавить новый тип слайда:
1. Сначала прочитай `CLAUDE.md` полностью
2. Внеси изменение в `CLAUDE.md` (правило) и в `build.py` (реализация)
3. Проверь, что изменение не ломает существующие слайды — запусти `python3 build.py`

**Нельзя менять без явного запроса пользователя:**
- Размер слайда (всегда 12192000 × 6858000 EMU)
- Цвета вне дизайн-системы
- Логотип и футер `zerocoder.ru` (они на каждом слайде через `add_chrome()`)
- Выравнивание иконок (только left-align, не по центру)

## 6. Подготовка к публикации на GitHub

Репозиторий: `annazvg05/presentation-builder` (ветка `main`)

### Стандартный цикл публикации

```bash
# 1. Проверить что изменилось
git status
git diff

# 2. Добавить нужные файлы (не добавлять .pptx и Готовые презентации/)
git add build.py CLAUDE.md README.md
git add .claude/skills/presentation-builder/skill.md
# Иконки и фото — только если добавились новые
git add "Иконки/новая_иконка.png"

# 3. Зафиксировать
git commit -m "описание изменений"

# 4. Опубликовать
git push origin main
```

### Что НЕ пушить

`.gitignore` уже настроен, но проверяй вручную:
- `*.pptx` — не пушим (большие бинарники, меняются при каждом запуске)
- `Готовые презентации/` — не пушим
- `.DS_Store` — не пушим
- `__pycache__/` — не пушим

### Проверка перед пушем

```bash
git diff --cached          # что войдёт в коммит
git log --oneline -5       # последние коммиты
gh repo view               # статус репо на GitHub
```
