# Zerocoder Presentation System — Инструкция для Claude

## Как работать с этим проектом

Анна даёт ТЗ → Claude читает CLAUDE.md → пишет/обновляет `build.py` → запускает `python3 build.py` → готов файл `zerocoder.pptx`.

Файл всегда перезаписывается. Никаких промежуточных файлов создавать не нужно.

## Технический стек

- **python-pptx 1.0+** + **Pillow** (уже установлены)
- Слайд: `12192000 × 6858000` EMU (13.33" × 7.5", 16:9 widescreen)
- Шрифт: **Montserrat** (встроен в имя, Google Slides подтянет автоматически)
- Логотип рендерится через Pillow из SVG-путей — не требует cairosvg

## Дизайн-система

### Цвета

```python
# Фон обычных слайдов (градиент сверху вниз)
BG1 = "14133D"   # тёмный верх
BG2 = "2C2265"   # середина
BG3 = "423388"   # фиолетовый низ

# Фон секционных/заголовочных слайдов (диагональный)
SEC1 = "5638B2"
SEC2 = "A493F9"

# Акцент
ACCENT = "4CE566"   # зелёный

# Карточки
CARD2  = "0F0E38"   # тёмный фон карточки (контрастирует с обоими концами градиента)
BORDER = "7066C4"   # светло-фиолетовая обводка карточек

# Текст
WHITE  = "#FFFFFF"  # заголовки, основной текст
SECOND = "#EAEAEA"  # вторичный текст (тело карточек, ситуация)
FOOTER = "#C8C0F0"  # zerocoder.ru (светлый лавандовый — читаем на #423388 фоне)
WARN   = "#FFB547"  # предупреждение (жёлтый)

# Бейдж
BADGE_BG  = "3B2B83"  # фон плашки "Кейс N"
BADGE_TXT = "#9581E4"  # текст плашки
BADGE_BRD = "9581E4"   # обводка плашки
```

### Типографика

| Элемент | Размер | Начертание | Цвет |
|---|---|---|---|
| Заголовок слайда (все кейсы) | 36pt | Bold | WHITE |
| Заголовок секционного слайда | 54pt | Bold | WHITE |
| "Ситуация:" (лейбл инлайн) | 20pt | Bold | ACCENT |
| Текст ситуации (инлайн) | 20pt | Regular | SECOND |
| Текст карточек этапов | 16pt | Regular | WHITE |
| Заголовок карточки промпта | 16pt | Bold | WHITE |
| Тело карточки промпта | 16pt | Regular | SECOND |
| Плашка "Кейс N" | 13pt | Bold | BADGE_TXT |
| Предупреждение (⚠) | 12pt | Regular | WARN |
| Заголовки колонок (слайд 4) | 17pt | Bold | — |
| Списки в колонках (слайд 4) | 16pt | Regular | SECOND |

> Правило: если текст не помещается — уменьшать на 2pt. Наложений быть не должно.

### Скругления

```python
PILL   = 50000   # 100% — таблетка (бейджи)
R_CARD =  6000   # ~8px — карточки (мягкое)
```

### Отступы и позиции (в дюймах × 914400 = EMU)

```
LM = 0.58"    ← левый и правый margin
TM = 0.48"    ← верхний margin (только для слайдов без бейджа)
CW = SW - 2*LM ← ширина контентной области
```

## Типы слайдов

### 1. Секционный / заголовочный слайд
Фон: градиент `#5638B2 → #A493F9` (диагональ), один большой текст по центру.

```python
build_section_slide(prs, "Текст слайда")
```

Использовать для: "Перейдем к практике", "Раздел 2", "Итоги" и т.д.

---

### 2. Кейс-слайд (с этапами)
Структура сверху вниз:

```
y=0.042"   ── тонкая зелёная полоска (акцент-бар, всегда)
y=0.50"    ── бейдж "Кейс N" (таблетка, #3B2B83, обводка #9581E4)
y=1.02"    ── заголовок (44pt или 40pt если длинный, до y≈2.77")
y=2.95"    ── "Ситуация: [текст]" инлайн, 20pt (до y≈3.83")
y=4.05"    ── три карточки с иконками (до y≈5.73")
y=5.92"    ── предупреждение ⚠ (жёлтый, 12pt)
y=7.20"    ── футер "zerocoder.ru" (центр)
            ── логотип (правый нижний угол)
```

```python
build_keis_slide(prs,
    badge="Кейс 1",
    title="Заголовок слайда",
    situation="Текст ситуации...",
    cards=[
        (ICO_BOLT,  "Этап 1"),
        (ICO_BAG,   "Этап 2"),
        (ICO_FIRE,  "Этап 3"),
    ],
    warning=True  # показывать ли предупреждение об удалении данных
)
```

---

### 3. Слайд с промптами
Структура:

```
y=0.95"   ── заголовок 44pt
y=1.95"   ── 3 карточки (высота 1.62" каждая, зазор 0.14")
            Каждая карточка: иконка (0.34") + заголовок промпта 16pt + текст 14pt
```

```python
build_prompts_slide(prs,
    title="Примеры промптов",
    prompts=[
        (ICO_BOLT,  "Название",  "Текст промпта..."),
        (ICO_BAG,   "Название",  "Текст промпта..."),
        (ICO_CHECK, "Название",  "Текст промпта..."),
    ]
)
```

---

### 4. Слайд сравнения (две колонки)
Структура: заголовок + две карточки рядом, внутри — плашка-заголовок + списки с иконками.

- Левая колонка: тёмный бейдж `#3B2B83`, текст `#9581E4`
- Правая колонка: зелёный бейдж `#4CE566`, текст `#14133D` (тёмный на светлом)
- ✅ Плюсы → иконка `галочка.png`
- ⚡ Минусы → иконка `молния.png`

```python
build_comparison_slide(prs,
    title="Заголовок сравнения",
    left_label="Вариант А",
    right_label="Вариант Б",
    left_pros=["пункт 1", "пункт 2"],
    left_cons=["пункт 1"],
    right_pros=["пункт 1", "пункт 2", "пункт 3"],
    right_cons=["пункт 1"],
)
```

## Правила иконок

| Иконка | Когда использовать |
|---|---|
| `крестик.png` | Минусы, риски, отрицание — **всегда** для "Минусы:" пунктов |
| `галочка.png` | Плюсы, завершение, результат, финальный этап |
| `молния.png` | Сбор данных, первый этап |
| `портфель.png` | Бизнес-документы, подготовка отчётов, второй этап |
| `огонек.png` | Анализ, инсайты, визуализация, третий этап |
| `рукопожатие.png` | Сотрудничество, команда, поддержка |

Не обязательно использовать все иконки. Выбирать по смыслу этапа/пункта.

## Обязательные элементы на каждом слайде

Функция `add_chrome(slide)` добавляет автоматически:
- Тонкая зелёная полоска сверху (`#4CE566`, 0.042")
- Футер `zerocoder.ru` по центру снизу (цвет `#C8C0F0`)
- Логотип zerocoder в правом нижнем углу

Эти элементы **нельзя убирать** ни с одного слайда.

## Фон карточек

Карточки используют `CARD2 = "0F0E38"` — намеренно темнее, чем `BG1 = "14133D"`.  
Это создаёт глубину: карточки "всплывают" над градиентом в обоих его концах (тёмный и фиолетовый).

## Ситуация — инлайн-формат

```python
txt_inline(slide,
    "Ситуация:",          # зелёный bold лейбл
    "Текст ситуации...",  # белый regular текст
    LM, y, CW, h, size=20)
```

Лейбл и текст находятся в **одном абзаце** (два run'а) — текст переносится единым блоком.

## Workflow при новом ТЗ

1. Прочитать ТЗ
2. Определить типы слайдов (секционный / кейс / промпты / сравнение / кастомный)
3. Добавить нужные функции в `build.py`
4. Обновить `main()` — добавить вызовы в правильном порядке
5. Запустить `python3 build.py`
6. Показать результат Анне

## Что нельзя делать

- ❌ Менять размер слайда (всегда 16:9 widescreen)
- ❌ Убирать логотип или футер
- ❌ Использовать другие цвета вне дизайн-системы (без явного запроса)
- ❌ Центрировать иконки в карточках (только left-align)
- ❌ Использовать `slide.background` для фона — не работает в Keynote/Google Slides. Всегда использовать `_apply_gradient()` (полноразмерный прямоугольник на заднем плане)
- ❌ Конвертировать SVG логотипа в другой формат вручную — он рендерится через Pillow в `_build_logo()`
